# 출결 팝업 저장 흐름 정리

## 목적

이 문서는 `일일출결입력` 팝업 안에서 다음 단계가 어떻게 안정화되었는지 정리한다.

1. `결석 > 질병` 라디오 선택
2. 사유 입력
3. 팝업의 `저장` 버튼 클릭
4. 저장 확인 팝업 표시

이 단계는 학생/날짜 칸을 여는 문제와는 별개로, 팝업 내부 조작을 안정화하는 작업이다.

## 실제 화면 기준으로 확인된 흐름

현재 화면에서 하루 전체 결석을 입력할 때는 학생 행의 `조회`, `1교시` 같은 시간 칸을 찍는 방식이 아니다.

- 학생 날짜 칸을 눌러 `일일출결입력` 팝업을 연다.
- 팝업 상단의 `결석` 구역에서 `질병` 동그라미를 찍는다.
- 아래 `사유` 입력 칸에 내용을 넣는다.
- 팝업 하단 `저장` 버튼을 누른다.
- 그 다음 `해당자료를 저장하시겠습니까?` 확인 팝업이 뜬다.

즉, 하루 전체 결석 입력은 `결석 라디오`를 찍는 것이 핵심이고, 학생 행의 시간 칸을 별도로 찍으면 안 된다.

## 질병 라디오를 누르는 방식

이 구간은 일반 텍스트 클릭으로는 잘 안 맞았다.

이유는 다음과 같다.

- `질병` 글자와 실제 클릭 영역이 분리되어 있다.
- 텍스트, 라벨, 아이콘이 커스텀 컴포넌트로 묶여 있다.
- 단순 `text=질병` 방식은 `결석` 행이 아니라 다른 위치와 섞일 수 있다.

현재 전략은 다음 순서로 동작한다.

1. 팝업 안에서 `결석` radiogroup을 찾는다.
2. 그 안에서 `질병` 항목을 찾는다.
3. 해당 항목의 실제 `radio icon` 위치를 DOM 기준으로 계산한다.
4. 계산된 좌표를 직접 클릭한다.

이 방식으로 바꾼 뒤 실제 `radio-test`에서 `결석 > 질병` 선택이 성공했다.

## 저장 버튼까지 가는 흐름

질병 선택이 끝나면 다음 순서로 진행한다.

1. `사유` 입력 필드를 찾는다.
2. 사유 문자열을 입력한다.
3. 팝업 하단의 `저장` 버튼 후보를 찾는다.
4. `저장` 클릭 뒤 확인 팝업을 기다린다.

여기서 중요한 점은 `팝업 저장`과 `페이지 최종 저장`을 구분하는 것이다.

- 지금 단계에서는 출결 팝업 내부의 `저장`만 누른다.
- 페이지 전체 저장이나 `마감` 단계는 아직 자동화 범위가 아니다.

## 저장 확인 팝업 처리 방식

팝업 저장 후에는 별도의 확인 팝업이 뜬다.

- 제목 또는 본문에 `해당자료를 저장하시겠습니까?`가 포함된다.
- 버튼은 `확인`, `취소`가 뜬다.

현재 자동화는 이 팝업이 뜨면:

1. 열린 dialog 목록을 다시 검사한다.
2. `확인` 버튼을 찾는다.
3. 해당 버튼에 클릭 이벤트를 보낸다.
4. 이후 원래 출결 팝업이 닫히는지 확인한다.

즉, 저장 성공 판단은 단순히 `저장` 버튼을 눌렀는지가 아니라:

- 저장 확인 팝업이 떴는지
- `확인`까지 눌렸는지
- 원래 출결 팝업이 닫혔는지

이 세 단계를 모두 통과해야 한다.

## 이번 단계에서 얻은 운영 원칙

- 시간 칸 입력과 하루 전체 결석 입력은 다른 흐름이다.
- 하루 전체 결석은 `결석 > 질병/미인정/기타/출석인정` 라디오를 사용한다.
- 팝업 저장 후에는 항상 확인 팝업이 뜰 수 있다고 가정해야 한다.
- 저장 확인 팝업은 본 처리 흐름의 일부로 본다.
